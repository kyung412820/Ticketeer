# Frontend State Strategy Patch

## Added
- `FRONTEND_STATE_STRATEGY.md`

## Why
프론트도 단순히 fetch만 붙인 것이 아니라,
- UI 상태
- 서버 상태
- optimistic update 회피
- React Query 도입 기준
까지 설명할 수 있어야 설계력이 보입니다.

## Portfolio Point
- UI state와 server state를 분리해서 설명 가능
- 왜 React Query를 아직 넣지 않았는지 설명 가능
- 왜 티켓팅 도메인에서 optimistic update를 조심했는지 설명 가능
